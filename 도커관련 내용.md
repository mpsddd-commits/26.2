# 콘테이너 관리할때 네트워크를 관리 하기 쉽게 하는게 아이디 대신 콘테이너 네임으로 연동가능하다! ex) curl http://web:80, localhost와 비슷한 느낌
 


# 도커는 기본적으로 컨테이너 볼륨 네트워크 3가지가 있다 생각하면된다.
# 도커 내부 볼륨은 도커 허브 내부에 저장되어 있다. 컨테이너 만들고 연결 해놓으면 볼륨은 관리 되어 있는데 컨테이너 지워져도 복구 가능하다. 볼륨관리는 내부 1 외부 1 두가지가 있다.

# 네트워크 종류가 많은데 그중 제일 중요한것은 브릿지이다

# 포트는 원래 같은 포트로 연동하면 안되는데 컨테이너끼리는 독립적이라 같은 포틑 사용가능하지만 pc와 도커 안의 네트워크와 연결할때는 하나씩만 사용 가능하므로 포트포워드는 외부 os와 도커호스트 네트워크 연동이다 컨테이너 내부 포트가 아니다. 그리고 내부포트는 같은 번호를 가질 수 있다 (왜냐하면 컨테이너 내부 포트는 같은 값을 줘도 되므로)
